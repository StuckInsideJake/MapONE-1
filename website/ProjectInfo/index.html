Introduction

The planetary science community develops, collects, and distributes planetary geologic mapping research. This research includes publications presenting mapping work and the development process of new map products. Similar to geologic mapping on Earth, planetary maps identify key features such as craters, canyons, and fault lines on other planets. Scientists use these shared resources within the community for numerous reasons. This includes mapping space exploration sites and learning about the elemental composition of different planets and other related scientific research. The client, the United States Geological Survey (USGS) Planetary Geologic Mapping (PGM) Program, assists the community by developing tools and resources to better access and use planetary data for these purposes. For example, PGM mapped the landing/exploration site for the National Aeronautics and Space Administration's (NASA's) Mars 2020 Perseverance Rover mission.

Problem

Scientists publish their planetary maps and research either through USGS or various online sources (often in journal articles and conference papers). Because USGS mandates certain map standards, many publications are distributed throughout the internet instead. Nevertheless, these resources are valuable to the client to provide the community with all types of publications. As shown in Figure 1, USGS currently does not have an automated way to collect these existing resources. Overall, the client and the planetary science community lack a single, centralized system for maps published in non-USGS venues.

Solution

The envisioned product, MapONE, is a user-friendly interface accessible through the USGS's website. The interface will display map metadata for a selected region of interest and connect to one database containing non-USGS publications. The project team will also create a web scraper to identify and collect these publications from online sources. Overall, MapONE will allow researchers to access non-USGS map metadata. As shown in Figure 2, the following technologies will be used to create this product:

  • Graphical User Interface (GUI): A GUI will be used to display map data on a single web application.
  • Web Scraper: The project team will develop a data extraction tool, known as a web scraper, to automatically detect and gather map publication metadata (author, source, region, etc.) from online non-USGS sources.
  • Central Database: MapONE will store publication metadata in one centralized database.
  • Remote Storage and Servers: Cloud computing services can be used to store map data and run the product's web application.
  • Web Framework: MapONE's web framework will filter metadata (author, source, region, etc.) from the database using user requests from the GUI.

Technologies

  • Flutter: Frontend web-based GUI.
  • Keras: Python-based machine learning library used for the web scraper.
  • Django: Full-stack web framework.
  • PostgreSQL: Database structure.
  • Amazon Web Services (AWS): Used to host the GUI and web framework.

System Requirements

Users will have access to the following features:

  • Login into an account on the USGS website.
  • View and filter planetary map metadata (author, source link, region, etc.).
  • Download all publication entries.
  • View and save search history results.
  • Automate searches periodically.
  • Receive notifications on new publications from automated search results.
